# 产品说明书 (Product Specification)

> **核心原则**：本文档是本项目的“单一事实来源 (Single Source of Truth)”。所有需求变更、架构调整、运维操作必须先更新本文档，再执行代码修改。

## 📚 目录导航

### 0. [AI 上下文指南](./00-context-for-ai.md)
* **用途**: 供 AI 助手快速理解项目背景、技术栈、代码规范和关键路径。
* **更新时机**: 项目结构变更、引入新技术栈时。

### 1. [需求说明书](./01-requirements.md)
* **用途**: 记录已实现功能、待开发需求 (Backlog)、业务逻辑规则。
* **更新时机**: 提出新需求、功能验收完成时。

### 2. [架构设计](./02-architecture.md)
* **用途**: 系统架构图、目录结构解析、关键数据流、组件设计。
* **更新时机**: 重构代码、修改核心组件、变更数据模型时。

### 3. [开发与运维手册](./03-workflows.md)
* **用途**: 环境搭建、开发命令、部署流程、内容发布指南。
* **更新时机**: 修改构建脚本、变更部署平台时。
